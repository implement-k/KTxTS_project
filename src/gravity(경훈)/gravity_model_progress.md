# Gravity Model Update


기존 코드는 내부통행과 외부통행이 명확히 분리되지 않은 상태에서 IPF를 적용할 수 있었고 거리행렬 대각선의 0 값이 `1e-3`으로 보정되면서 자기동 통행에 과도한 거리저항 가중치가 생길 수 있었다. 그래서 외부통행은 IPF로 배분하고, 내부통행은 별도 LightGBM 모델로 예측한 뒤 최종 OD 행렬의 대각선에 삽입하도록 변경했다.

수정 내용:

- `O_pred`, `D_pred`는 내부통행을 제외한 외부 유출·유입 총량으로 사용
- 외부 OD 배분 시 거리저항 행렬의 대각선을 `0`으로 처리 (그니까 내부통행을 0으로)
- 내부통행 `y_self`는 별도 예측 후 최종 OD 대각선에 삽입
- IPF 수렴 확인을 `A`, `B` 계수 변화가 아니라 실제 행합/열합 오차 기준으로 변경
- 테스트 구역 행정동 코드를 `OD_dong_list.xlsx` 기준으로 수정하여 동탄, 위례, 검단 총 17개 행정동이 포함되도록 변경

현재 기준 결과 (`beta=1.5`):

- 전체 OD RMSE: `274.3226`
- 전체 OD CPC: `0.7067`
- Test RMSE: `525.7611`
- Test CPC: `0.5744`

## Beta Tuning

beta는 거리저항 계수로, 값이 커질수록 가까운 행정동으로 더 많이 배분된다. 즉 beta가 작으면 통행량이 장거리까지 넓게 퍼지고, beta가 커지면 근거리 행정동에 더 강하게 몰린다.
다만 beta를 너무 크게 설정하면 장거리 통행을 과소배분할 수 있다. 따라서 현재는 단일 beta 값만 조정하여 실험했고, 이후에는 거리 구간별로 별도 가중치를 두는 방식도 검토할 예정이다.

실험한 beta 값:

- `1.0`
- `1.5`
- `2.0`
- `2.25`
- `2.5`

요약 결과:

| beta | 전체 RMSE | 전체 CPC | Test RMSE | Test CPC | 1000+ RMSE | 1000+ CPC |
|---:|---:|---:|---:|---:|---:|---:|
| 1.0 | 328.1064 | 0.6068 | 541.0951 | 0.4960 | 6881.6599 | 0.4714 |
| 1.5 | 274.3226 | 0.7067 | 525.7611 | 0.5744 | 6682.7498 | 0.5606 |
| 2.0 | 237.6193 | 0.7745 | 515.0630 | 0.6197 | 6506.8599 | 0.6324 |
| 2.25 | 247.7306 | 0.7780 | 516.0963 | 0.6219 | 6480.9594 | 0.6490 |
| 2.5 | 277.7205 | 0.7638 | 522.4568 | 0.6123 | 6512.8744 | 0.6551 |

현재 해석:

- `beta=2.0`은 전체 RMSE와 Test RMSE 기준에서 가장 안정적이다.
- `beta=2.25`는 Test CPC와 1000명 초과 OD 구간 성능이 조금 더 좋다.
- `beta=2.5`는 근거리 OD에는 강하지만, 장거리 OD를 과소배분할 가능성이 있다.

따라서 현재 후보는 `beta=2.0` 또는 `beta=2.25`이다. 이후 거리 구간별 성능과 주요 장거리 OD를 추가로 확인한 뒤 최종 beta를 선택할 예정이다.
